# Aki Adaptive Script Rules

## Core Direction

1. 爆点直给，不先讲背景。
2. 前 3-5 秒必须回答“为什么值得关注”。
3. 用短句推进：一行一个信息点。
4. 给判断，不只复述事实。
5. 更像人对镜头讲，不像把文章切碎后分行。

## Tone

1. 可以锋利，但不喊口号。
2. 禁止“模板腔开场”与空洞 CTA。
3. 对比句可以使用，但 `不是…而是…` 这类结构全篇最多 1 次。
4. 相邻三句不得重复同一种骨架句式（如连续使用“不是…而是…”、“真正重要的不是…而是…”）。
5. 允许一点“口语提纲感”，但整体仍然要能直接念。

## Plain Language First

1. 优先用普通人一听就懂的话，不要把专业感当高级感。
2. 能说“步骤很多、要连续做很久的任务”，就不要写“长链路任务”。
3. 能说“把网页、文档、代码放一起做”，就不要写“工具协同升级”。
4. 除非品牌名本身就是重点，否则不要让英文名、缩写词、专业术语占据句子的中心位置。
5. 不要用空泛强词替代信息，比如：`更稳了`、`都能顶住`、`全面升级`、`更丝滑`。如果写提升，必须顺手说清楚提升在哪。
6. 不要把口播写成“概念解释”。先讲人能感受到的变化，再讲技术词。

## Structure by Duration

1. 不固定三点，不固定段数。
2. 默认优先拆成 `2-4 点`，但如果题目简单，可以只展开 `1-2 点`。
3. `15-30s`：强爆点 + 1-2 个展开点 + 判断收束。
4. `31-90s`：爆点 + 2-3 个展开点 + 代价/价值 + 立场。
5. `91-300s`：开场判断 + 2-4 个展开点 + 最终立场。
6. 允许出现 `详细说说：`、`一. 二. 三.` 这种口语提纲感骨架，但只在帮助表达时使用，不做强制模板。

## Hard Bans

1. 禁用：`你是否...`、`我已经...`、`快去试试`。
2. 禁用“先定义概念再开始”的拖沓写法。
3. 禁用“正确的废话”。
4. 禁用标题、时长、`先说结论` 这类说明性前缀。
5. 禁用把文章里的 markdown 小标题、分隔线、列表直接搬进口播稿。
6. 禁用高频堆叠 AI 对照模板（如连续多次“不是 A，而是 B”）。
