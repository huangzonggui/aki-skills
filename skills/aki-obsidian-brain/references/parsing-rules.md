# 自然语言解析规则

## 路由优先级

1. 追溯：包含 `追溯选题`
2. 查询：`最重要的待办` / `最重要的todo` / `还有哪些没做` / `待办还有哪些` / `可写选题` / `还有哪些话题没写`
3. 记录：`记任务` / `记待办` / `todo` / `图度` / `记灵感` / `记想法` / `随手记` / `记选题`
4. 无法确定：返回 `need_confirmation`

## 路由边界

- 第一版只自动写入 `任务清单.md`、`灵感池.md`、`选题库.md`
- “观点”“个人生活”类输入默认不自动猜测，先确认
- 不按输入端分流
- 不写 `飞书`、`OpenClaw` 等来源标识
- 不创建 `移动采集_*` 或单独收件箱文件

## 任务象限映射

- Q1: 重要且紧急 / 紧急且重要
- Q2: 重要不紧急 / 不紧急但重要
- Q3: 紧急不重要 / 不重要但紧急
- Q4: 不紧急不重要
- 无明确映射：默认 Q2

## 选题字段提取

- 标题：关键词后正文，去掉 `来源`、URL、`动机`字段。
- 来源：提取第一个 URL（可选）。
- 动机：匹配 `动机: ...`（可选）。

## 去重规则

- 标题精确匹配即视为同一选题。
- 同标题新输入不新增条目，只补充来源/动机。
