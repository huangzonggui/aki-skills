# Aki 30s Video Script Style

按下面流程写，不要套通用 AI 口播模板。

## Workflow

1. 先抽一句核心观点（必须可争辩、可判断，不是中性描述）。
2. 先写“前5秒爆点句”：
   1. 明确这个东西“为什么现在值得关注”
   2. 优先写热度数字 + 冲突反转（如爆火/下架/封禁/争议）
   3. 没有热度数字时，用“结果级变化 + 代价”替代
3. 再定 4 段式骨架：
   1. 爆点：5秒内说出关注理由（热度/冲突）
   2. 机制：解释“为什么成立”
   3. 价值：说明长期意义或行业后果
   4. 收束：一句判断句落地
4. 从公式库里选 1 个骨架再写（默认 `S5-HMVC`）。
5. 输出单一终稿；未被要求时，不提供多版本。

## Formula Library

### S5-HMVC（默认，Aki优先）

- `S5`（0-5秒）: 值得关注的爆点（热度数字 + 反转冲突）
- `H`（5-12秒）: 这个事情到底是什么
- `M`（12-20秒）: 它背后的机制
- `V`（20-27秒）: 长期价值/行业影响
- `C`（27-30秒）: 一句判断收束

### HBC（TikTok官方三段式）

- `Hook` -> `Body` -> `Close`
- 适合：强传播、强转化、信息点较少的题材

### PAS（Problem-Agitate-Solution）

- `Problem` -> `Agitate` -> `Solution`
- 适合：痛点型、工具型、方法型内容

### BAB（Before-After-Bridge）

- `Before` -> `After` -> `Bridge`
- 适合：对比转变型、认知升级型内容

## Length

- 30 秒：130-190 个中文字符（不含标题）
- 20 秒：80-130 个中文字符
- 15 秒：55-95 个中文字符

## Voice Rules

- 用短句，1 句只承载 1 个信息点。
- 优先判断句，不写空泛抒情。
- 多用“不是...而是...”“真正的关键是...”“更重要的是...”这类结构。
- 尽量给具体名词或数字，不写抽象大词堆砌。
- 可以有锋芒，但不要喊口号。

## Hard Bans

- 禁用模板腔：`你是否...`、`我已经...`、`这也是为什么...`
- 禁用空 CTA：`快去试试`、`赶紧收藏关注`
- 禁用“先铺垫背景再说观点”的写法
- 禁用“前5秒还在定义概念”的写法
- 禁用“正确的废话”：如“AI改变世界所以我们要学习AI”

## Output Format

默认输出：

```markdown
# 30秒口播脚本
<正文，4-6行短句>
```

只要用户要求“纯口播”，就不要加分镜、时间轴、舞台提示。

## Self-check (must pass)

- 第一行是否能单独当标题传播？
- 前5秒是否已经回答“为什么现在值得关注”？
- 是否明确使用了某一个公式骨架（不是混乱拼接）？
- 是否在前两句内讲清“核心机制”？
- 是否有一句“价值判断”而不是事实复述？
- 删除任意一句后，信息是否会缺失？如果不会，删掉它。
